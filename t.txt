v0.1 shake-down: responsive examples, 3 fixes, blink caret, minmax()

Drove every example on screen and fixed the bugs that only show up there.

Examples
- list/gallery made responsive (width:100% + max-width); gallery is now a
  flex-wrap grid; dashboard rebuilt as a dark minmax(0,1fr) stat grid.
- battery kept deliberately fixed-width as the control.

Fixes (all test-invisible, found by driving the app)
- flex-wrap + percentage width + max-width mis-measured its own height: the
  container reserved one row while painting two, so the next sibling rode up
  over the wrapped row. Root cause is a taffy bug still present in 0.12; for
  that exact combo we now map the width to `auto` (fit-content, still capped).
  Regression test in rux-layout/tests/wrap.rs.
- @tap inside r-for couldn't see the loop variable: handlers run later in
  global scope where the r-for binding is gone, so `picked = item` silently
  did nothing. Bake the loop bindings into the handler as a `let` prelude at
  build time (bind_locals + Value::to_rhai_literal); eval's existing rewind
  drops them, so nothing leaks. End-to-end test in rux-style.
- fixed-width fields/rows overflowed narrow windows; made them responsive.

Features
- Blinking caret: 530ms, solid while typing, timer only runs while an input is
  focused so an idle window stays fully event-driven (rux-shell + a
  caret_visible gate in rux-paint::build_scene, no rebuild).
- minmax(0, 1fr) grid tracks: new Track::MinMax -> taffy minmax(), paren-aware
  parse_tracks, so grids shrink below content instead of overflowing.

Verified: minimize/restore clean, hot-reload live, 24 test suites green.
Still open before tagging v0.1: cross-DPI drag (ScaleFactorChanged), untested.
